# Wen13 世界物件落位规则

这个项目不是可自由环绕的第三人称相机。默认游戏镜头在 `View.js` 中固定为 `theta = π × 0.25`，也就是从世界坐标的 `+X / +Z` 方向斜俯视目标；鼠标或触控拖动只移动观察中心，不会绕物件旋转。

因此，人物、雕像、招牌、屏幕、文字、按钮和有明显正反面的模型，不能只检查世界坐标，还必须检查“主镜头可读面”。

## 落位闸门

1. 在 Blender 中明确记录模型的本地正面轴，但最终以游戏实景截图为准，不能只凭建模软件坐标猜测。
2. 让正面朝向默认镜头的 `+X / +Z` 象限；骑士从原先背面的 `Z = 180°` 调整为 `Z = 0°`，在固定镜头下形成可读的三分之四视角。
3. 用桌面默认镜头、手机窄屏和昼夜至少三个状态截图，确认正面、轮廓与交互提示都可读。
4. 驾车从主要路线经过一次，确认物件没有被树、字标、建筑或 UI 长期遮挡。
5. 如果一个物件必须从背面也读得懂，应做双面材质、镜像副本或四向造型，不能依赖用户“转视角”。

## 可复用原则

- 雕像/人物：正面朝 `+X / +Z`，优先呈三分之四视角。
- 招牌/屏幕：读面法线朝 `+X / +Z`；必要时做双面显示。
- 装饰物：可无方向，但轮廓必须从固定俯视角成立。
- 新区域：先放相机参考锥体，再批量布置有方向的资产。

骑士雕像的导出脚本和 `validate:landing` 已把朝向与手机可见落位固化；若未来误转回背面或移回窄屏画面外，自动校验会失败。
