## 1. 本轮结论

本轮选择 **方案一：真实名人样板**，首位候选确定为 **韩寒**。

这不是因为“名气越大越好”，而是因为他的公开职业轨迹天然形成了三个可验证、视觉语言又互相咬合的身份：**作家、职业赛车手、电影编剧/导演**。它们可以直接映射到现有网站的 Career、Circuit、Vehicle、Projects、Lab、Time Machine 和 Landing 雕像，绝大多数适配不需要改变地图和玩法。

本样板要验证的不是能否制作一个“名人粉丝站”，而是以下服务能力：

1. 能否把公开履历编辑成一个不扁平的人物命题；
2. 能否把事实证据、设计解释和艺术化演绎清楚分开；
3. 能否用同一套视觉系统统一文字、UI、车皮、场景贴图与雕像；
4. 能否只复用现有世界，在 L0–L2 范围内做出明显属于这个人的版本；
5. 能否沉淀成以后面对真实客户时可复制的工作流。

> **条件性入选说明**：韩寒在内部样板的综合适配评分中最高，但“权利可控性”只有 1.5/5，低于工作流对公开客户版本要求的 3/5。因此他只通过“内部研究候选”闸门，**没有通过公开或商业生产闸门**。后续 Gate C 也只能制作受控的抽象视觉样张；若要公开上线，必须先取得本人及相关作品、肖像与商标权利人的授权，或改用已授权客户/完整虚拟人物。

### 1.1 为什么不是另外两个候选

| 候选 | 强项 | 本轮不选的原因 |
|---|---|---|
| 董宇辉 | 公开表达、教育转型、文化旅行资料较多，时间线清楚 | 内容更依赖演讲、知识服务和真实媒体素材；对现有赛车场、车辆、时间机器的个性化复用较弱 |

## 3. 人物核心命题

### 3.1 对外叙事标题

**《两种速度》——在文字、赛道与镜头之间**

### 3.2 一句话人物命题

> 一位不断把观察转成文字、把判断放上赛道、再把速度组织成电影叙事的跨媒介创作者。

这句话属于 **I1 编辑性解释**，不是本人引语；正式客户项目必须经本人确认后才能成为首页文案。

### 3.3 访问者离开时应记住的三件事

1. 他不是“写书之余玩赛车”，赛车是经过官方成绩验证的第二职业；
2. 电影不是突然换行业，而是把文字叙事、道路经验和速度感合并到一个新媒介；
3. 他的重要特征不是标签多，而是愿意进入不同专业，并接受各自的规则与检验。

### 3.4 核心张力

**慢速编辑与高速判断**：写作、剪辑需要反复推敲；赛车的决定却必须在瞬间完成。整个网站的动线和视觉都围绕这组张力组织，而不是简单堆放书、赛车与摄影机。

### 3.5 文案口吻

- 关键词：克制、具体、略带自嘲；
- 禁用词：传奇、天才、封神；
- 禁止把人写成没有失败、没有团队、没有时间变化的“成功学角色”；
- 所有数字、名次和职务都连接来源；设计解释主动标注“策展观点”或“艺术演绎”。

## 4. 体验动线：一圈看懂三种身份

### 4.1 首次探索动线

```text
入口雕像
  ↓ 先看到“人”与核心命题
少年写作 / Career
  ↓ 理解公开声音从哪里开始
职业赛道 / Circuit + Vehicle
  ↓ 通过驾驶亲自感受规则、判断与失误
项目看板 / Projects
  ↓ 用四个证据故事确认写作、赛车、电影的变化
方法实验室 / Lab
  ↓ 看见尚未封闭的问题，而不是另一份作品清单
时间机器 / Time Machine
  ↓ 对照 17 岁、职业转折、导演起点与当下
通用联系 / Social
  ↓ 保持几个占位入口，不承担人物叙事
```

### 4.2 直达动线

菜单和地图必须同时提供“人物简介、四个项目、时间线、Lab、来源与演绎说明”的文字入口。不愿开车、使用手机或性能较弱设备的访问者，也能在不进入玩法的情况下理解完整人物。

### 4.3 动线原则

- Landing 只回答“为什么值得继续看”；
- Career 讲变化，不堆作品；
- Projects 讲本人角色、关键决定和证据；
- Circuit 让访问者体验规则与判断，不假装复刻真实赛事；
- Lab 讲尚未结束的问题；
- Time Machine 做版本对照，不重复 Career；
- Social 只留通用模型，不做账号深度集成。

## 7. Projects：四个旗舰故事

### 7.1 项目 01：少年写作——公开声音如何诞生

**核心问题**：一个 17 岁写作者怎样从强烈表达走到长期创作？
**本人角色**：作者。
**事实锚点**：早期写作、首部长篇、后来公开反思年轻时期的表达变化。
**页面六幕**：写作处境 → 首次被看见 → 长篇写作 → 早期锋芒 → 自我修正 → 仍然写作。

三张画面：

1. `P01-EVIDENCE`：1999–2003 的原创出版时间轴，只使用文字、页码、年份和来源脚注；
2. `P01-EXPLAIN`：一页文本经过“观察—句子—段落—公共回应—重新编辑”的原创流程图；
3. `P01-ATMOSPHERE`：纸页像公路一样向远处展开的低多边形场景，标注“艺术演绎”。

禁用：原书封、书页扫描、长段引文、仿手稿、销售数字的无源宣传。

### 7.2 项目 02：职业赛车——从被当作玩票到年度双冠

**核心问题**：跨界者如何证明自己尊重另一个专业？
**本人角色**：职业车手；团队贡献必须保留。
**事实锚点**：2003 职业起点；2012 年拉力和房车赛年度冠军；官方奖项。
**页面六幕**：第一次参赛 → 训练和车队 → 两类赛制 → 赛季挫折 → 2012 双冠 → 对规则的尊重。

三张画面：

1. `P02-EVIDENCE`：2012 两条官方成绩轨迹与来源，不绘真实赛车；
2. `P02-EXPLAIN`：拉力赛与场地赛的差异图——路书/计时对比走线/圈速；
3. `P02-ATMOSPHERE`：雨后沥青上一条红色理想走线，标注“艺术演绎”。

禁用：真实车队标识、赞助商、车号、赛车照片、赛事直播截帧、把团队成绩个人化。

### 7.3 项目 03：第一次执导——把文字变成镜头

**核心问题**：已成名的作者怎样重新成为一个新专业的初学者？
**本人角色**：编剧/导演，具体职务以公开片尾或权威资料核对。
**事实锚点**：2014 年导演转向与公开采访。
**页面六幕**：早年愿望 → 故事提案 → 从文本到分镜 → 团队协作 → 首次完成 → 新身份成立。

三张画面：

1. `P03-EVIDENCE`：2013–2014 的角色与生产阶段时间线，不使用影片画面；
2. `P03-EXPLAIN`：同一句原创占位文本如何转成镜头表、场面调度和剪辑节奏；
3. `P03-ATMOSPHERE`：三张漂浮分镜框沿公路排布，标注“艺术演绎”。

禁用：电影剧照、海报、角色肖像、对白、片方 Logo、未经许可的幕后照。

### 7.4 项目 04：赛车电影——两条职业线终于汇合

**核心问题**：真正懂赛车的人，怎样让观众在银幕上感觉到速度与比赛策略？
**本人角色**：编剧/导演；赛车知识作为创作背景，不把整个剧组贡献归于个人。
**事实锚点**：2019–2026 赛车题材电影的连续创作；2026 年公开谈到赛道设计和拍摄技术。
**页面六幕**：两种职业汇合 → 速度不是模糊 → 赛道承载叙事 → 拍摄技术 → 团队工业能力 → 下一圈。

三张画面：

1. `P04-EVIDENCE`：作品年份、公开职务和权威来源形成一条中性时间线；
2. `P04-EXPLAIN`：原创“赛道几何—角色选择—摄影机路径—剪辑节奏”四层图；
3. `P04-ATMOSPHERE`：一条赛道轨迹逐渐变成胶片/时间轴的抽象图，标注“艺术演绎”。

禁用：片名 Logo 视觉化、海报构图、人物造型、真实赛道/赛车复制、票房作为个人能力的单一证据。

## 8. Lab：速度编辑室

Lab 不是“韩寒未公开想法猜测区”，而是一组由公开工作材料引出的策展问题。每张卡片必须显示 `公开材料`、`策展问题` 或 `艺术演绎` 标签。

| Lab 卡片 | 它展示什么 | 可视化方式 | 声明 |
|---|---|---|---|
| 句子怎样成为镜头 | 文本到分镜、调度、剪辑的转换 | 可拖动的四阶段管线 | 策展解释 |
| 速度怎样被看见 | 机位、镜头路径、路面参照物与剪辑 | 速度感构成分解图 | 基于公开制作资料 |
| 两种赛制，两种判断 | 拉力与场地赛对驾驶判断的不同要求 | 左右分屏对照 | 常识＋公开成绩背景 |
| 路线就是叙事 | 赛道如何承载选择、悬念与超越 | 叙事节点叠加赛道图 | 基于 2026 公开发言的编辑转译 |
| 失败怎样进入喜剧 | 不复制影片情节，研究失败、等待和重来的节奏 | 抽象节拍器与转折曲线 | 艺术演绎，不归因本人观点 |
| 下一次起跑 | 当前仍在推进的创作只显示公开状态 | 未闭合的圈与日期 | 事实到此为止，不预测内容 |

Lab 的主图仍使用当前 960×540 规格，缩略图从同一源文件安全裁切。所有图形使用真实文字数据和原创几何，不生成假现场。

## 9. Landing 雕像设计合同

### 9.1 概念

雕像名称：**“带着头盔的编辑者”**。

它不是把三种职业做成三个 Logo，而是用一个姿态表达两种速度：人物站立，身体略向前；左臂夹一只无品牌头盔，右手拿一组被风掀起的纸页/分镜卡。底座上的刻线一侧像校对符号，另一侧逐渐变成赛道走线。

### 9.2 风格与肖像等级

- 低多边形、非写实、略带石膏模型与纸张切面的质感；
- 内部概念阶段只做到“成年人、短发、创作者/车手”的抽象相似，不追求面部复刻；
- 获得书面肖像授权后，才进入可识别五官、服装与个人特征版本；
- 不使用电影角色造型、赛车服商标、品牌头盔或签名。

### 9.3 固定镜头构图

- 雕像正面三分之四朝向默认相机与主行车线，不能沿 Blender 正视图自行判断；
- 入口出生点、向左/向右各行驶一个车身距离、地图传送落点各做一次截图；
- 头盔轮廓、脸部朝向、手中纸页必须在固定斜俯视视角中同时可读；
- 夜间用底座暖光勾边，不给脸部做写实聚光；
- 碰撞体只包住底座和主体，不让纸页等突出件卡车。

### 9.4 技术合同

| 项目 | 要求 |
|---|---|
| 文件 | 保持独立 `GLB`；不嵌外部版权纹理 |
| 根节点 | 继续满足现有 `landingKnightStatuePhysicalFixed` 识别契约，或在代码中以显式新名字同步修改并加验证 |
| 尺寸 | 以现有入口雕像包围盒为基准，不改变出生区可驾驶宽度；视觉高度约为车辆的 3–4 倍 |
| 几何 | 建议 8k–20k triangles；轮廓优先，不为看不见的背部堆细节 |
| 材质 | 2–3 个材质：纸石、石墨、信号红；避免高成本透明和复杂节点 |
| 纹理 | 能用顶点色则不用纹理；必须使用时合并为单张小型图集并生成压缩版本 |
| 动画 | v0.1 不做骨骼和动作，只靠姿态、纸页角度与灯光表达 |
| 物理 | 固定刚体、简化碰撞；睡眠启用；不得改变玩家车辆物理 |
| 文件预算 | 目标压缩后不高于现有约 197 KB 雕像的 2–3 倍；超出必须说明收益 |

## 10. 全站视觉系统

### 10.1 视觉母题

**纸张 / 沥青 / 校对线 / 理想走线 / 铝 / 钨丝灯**。

写作区使用偏暖纸白和细校对线；赛车场使用沥青、石墨和信号红；电影与 Lab 用铝灰和画幅框连接两者。所有区域共享同一种红色和同一种浅底色，避免每个项目像不同设计师临时做的海报。

### 10.2 颜色令牌

| Token | 色值 | 角色 |
|---|---:|---|
| `ink-950` | `#171716` | 主文字、夜间底色、轮胎/石墨 |
| `paper-100` | `#E9E0CF` | 面板、雕像纸石、日间主底 |
| `paper-300` | `#C8BDAA` | 次级面、旧纸感，不做怀旧滤镜 |
| `asphalt-700` | `#4B4C49` | 道路、二级标题、车辆主体 |
| `aluminium-400` | `#A8AAA5` | Lab 框架、电影生产语言 |
| `signal-red-500` | `#E43C32` | 唯一高强调色、校对线、理想走线 |
| `start-amber-400` | `#F0A33A` | 起跑灯、夜间提示、交互反馈 |
| `night-blue-950` | `#141A20` | 夜间背景，不用纯黑吞掉模型轮廓 |

### 10.3 字体与信息层级

- 中文正文与 UI：项目已使用的 Noto Sans SC；
- 大标题可使用同字体重字重，通过字距和切线建立个性，暂不增加第二套大体积中文字体；
- 年份与计时使用等宽数字风格，但不把全站做成赛车仪表盘；
- 每个项目画面右下角固定显示来源编号或 `艺术演绎`；
- 禁止把英文大字当装饰主体，中文版从第一轮就按中文长度设计。

### 10.4 统一规则

1. 所有看板使用 8pt 栅格、同一圆角/切角体系和同一边框粗细；
2. 一张画面最多一个红色重点，避免“到处都在起跑”；
3. 事实图用实线，策展解释用虚线，艺术演绎用无数据的面/光；
4. 日间对比度不靠阴影堆层；夜间核心文字、雕像正面和路径必须仍清楚；
5. 车辆、雕像、项目图、地图图标和 UI 使用同一套色票，不引用电影或车队配色。

## 12. 图像生产质量标准

### 12.1 三类图像必须各司其职

| 图像类型 | 回答问题 | 允许内容 | 不允许内容 |
|---|---|---|---|
| 事实证据图 | 发生过什么、何时、本人什么角色 | 日期、职务、结果、来源、原创数据结构 | 假现场、无源数字、AI 补出的“史料” |
| 原创解释图 | 这件事怎样运作、为什么重要 | 流程、关系、比较、方法、原创分镜 | 把编辑观点写成本人结论 |
| 氛围身份图 | 这段经历给世界什么情绪和记忆 | 抽象空间、材质、光、象征物 | 仿电影海报、仿真实照片、未标注演绎 |

### 12.2 单图验收

- 3 秒内能读出一个中心信息；
- 手机宽度下标题和主要数字不糊；
- 来源或演绎标签无需放大即可辨认；
- 不依赖品牌 Logo 才能认出主题；
- 不出现伪造的人脸、新闻截图、签名、证书、奖杯或项目界面；
- 同一项目三张图有叙事递进，不是三张同风格壁纸。

## 13. 实施顺序与验收闸门

### Gate A：事实与权利冻结

- 候选研究报告完成；
- 事实账本中所有主叙事条目至少为 F1/F2；
- 所有图片、模型、字体、Logo、名称有许可或原创策略；
- 名人样板明确为内部概念，不进入公开站点。

通过后才进入内容生产。

### Gate B：叙事与线框冻结

- 人物命题、四个项目、六个 Lab、八个 Career 节点不互相重复；
- 每个区域只回答一个问题；
- 桌面驾驶动线与手机直达动线均完整；
- 用户确认“这个方向值得作为首个样板”。

通过后才生产最终视觉。

### Gate C：视觉样张冻结

先制作而不是一次铺满：

1. Landing 雕像灰模三视图＋默认相机截图；
2. 项目 02 的事实图、解释图、氛围图各一张；
3. 一张 Career 石板；
4. 日间/夜间各一张 UI 合成图；
5. 车辆与赛道换皮小样。

这五组样张共同通过后，才批量完成其余资产。

### Gate D：集成与回归

- 修改前记录 Git 提交或可恢复快照；
- L0 数据、L1 贴图/UI、L2 模型分别提交，便于独立回退；
- `npm run validate`、`npm run build` 全部通过；
- Landing、Career、Projects、Lab、Time Machine、Circuit 和 Social 逐区检查；
- 固定镜头、桌面、手机、日间、夜间、完整 5 分钟昼夜循环做 QA；
- 驾驶、碰撞、翻车重置、传送、赛道计时和社交占位链接无回归。

### Gate E：发布决策

- 内部样板：只保留本地或受控预览；
- 对外方法展示：使用虚拟人物或获得明确授权的客户素材；
- 名人版本：必须有肖像、名称使用、作品/项目素材和可能的商标授权，且清楚声明制作关系；
- 未满足授权时，不得覆盖当前公开的 Wen13 版本。
